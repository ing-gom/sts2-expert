# STS2 카드 티어 판정 스킬

`data/card_tier_data.csv`의 카드 티어를 판정·조정할 때 사용.

## 티어 정의

| 티어 | 의미 | 기준점 |
|---|---|---|
| S | 게임 체인저 / 빌드 코어 | 큐레이션으로만 부여. 자동 승격 금지 |
| A | 매우 강한 픽, 범용성 높음 | 고효율 + 강한 효과 또는 조건부 레어 |
| B | 덱에 따라 유용 | 기본선. 평균 효율 |
| C | 조건 갖춰졌을 때만 유용 | 1코 6딜(Strike) 기준 |
| D | 클로그/명백한 함정 | 사용자가 명시한 경우만 |

## 1. 순수 스탯 효율 (베이스라인)

데미지/블록 값만으로 판단하는 기준선. 부가 효과 없는 `피해를 N 줍니다` / `방어도를 N 얻습니다` 단일 효과 카드에 적용.

| 코스트 | D | C | B | A | S |
|---|---|---|---|---|---|
| 0코 | <2 | 2-3 | 4-5 | 6-7 | 8+ |
| 1코 | <6 | 6-7 | 8-9 | 10-12 | 13+ |
| 2코 | <9 | 9-11 | 12-14 | 15-19 | 20+ |
| 3+코 | <10 | 10-15 | 16-21 | 22-27 | 28+ |

**하이브리드(데미지+방어도)**: `max(dmg,blk) + 0.5*min(dmg,blk)`로 합산.

**다중 타격은 실제 적힌 수치로만** — `~회`를 곱해 과대평가 금지. 게임 내 실제 수치(description에 명시된 최댓값) 사용.

## 2. 부가 효과 조정

각 효과는 ±티어.

### 긍정 (+1 per effect, 각 카드 최대 +3)
- **DRAW**: `카드를?\s*\d+장\s*뽑`, `카드를?\s*뽑`
- **AOE**: `모든\s*적`, `전체\s*적`
- **DEBUFF**: `취약`, `약화`
- **ENERGY_GAIN**: 에너지 아이콘 + `얻` (무조건인 경우만)
- **SCALING**: `영구적으로`, `힘을?\s*\d+\s*얻`, `민첩을?\s*\d+\s*얻`
- **HEAL**: `체력을?\s*\d+\s*회복`
- **ANTI_DEBUFF**: `적이\s*힘을?\s*\d+\s*잃` (적 디버프, 자해 아님)

### 약한 긍정 (+0.5)
- **RETAIN**: `다음\s*턴\s*시작\s*시.*손`, `보존`
- **SEARCH**: 덱/버린 더미에서 카드 가져오기
- **STAR_SCALING** (Regent): 별당 스케일링

### 부정 (-1 per effect, 각 카드 최대 -2)
- **DELAYED**: `\d+\s*턴\s*후`, `\d+\s*턴\s*뒤` — 예: 폭탄(3턴 후 피해)
- **FATAL**: `즉사` — 예: 포석(피해 입으면 즉사)
- **SELF_DMG_HEAVY**: `체력을?\s*[3-9]\s*잃`, `체력을?\s*\d\d+\s*잃`
- **SELF_DEBUFF**: `(?<!적이\s)힘을?\s*\d+\s*잃` — 적이 접두 없는 자해
- **LOSE_RES**: `밀집을?\s*\d+\s*잃`, `집중을?\s*\d+\s*잃` (Defect 리소스 손실)
- **PILE_HARD**: `소모된\s*카드`, `소모\s*더미에서` — 소모 더미 의존
- **HAND_DEP**: `손에\s*있는\s*.*카드`, `손의\s*카드` (특정 손 상태 요구)
- **HP_COND**: `체력이?\s*\d+\s*(이하|이상|미만)` — 체력 조건부 발동

### 약한 부정 (-0.5)
- **CONDITIONAL**: `했다면`, `있다면`
- **NEXT_TURN_DELAY**: `다음\s*턴에,`

## 2a. 소멸(EXHAUST) 페널티 (강한 -1 티어)

`소멸\.` 패턴 — 한 전투에 한 번만 사용 가능. **기본적으로 -1 티어**.

### 유지 조건 (시너지/전환 가치 있으면 예외)

소멸을 상쇄할 만큼 강한 가치가 있는 경우에만 A/S 유지:

| 전환 가치 | 예시 | 판정 |
|---|---|---|
| 에너지 대량 획득 | 생산(0코 2에너지+소멸), 2배 에너지, 에너지 쇄도 | A 유지 |
| 대량 드로우 | 전략의 천재(4장+소멸), 재기동(6장+소멸) | A/S 유지 |
| HP→자원 전환 | 사혈(3HP→2에너지), 제물(6HP→2에너지+3드로우+소멸) | A/S 유지 |
| 초강력 디버프 | 어둠의 족쇄(적 힘 9/15 감소+소멸), 충격파+(약화+취약 5 AOE) | A 유지 |
| 선천성+즉발 | 극적인 입장(0코 선천 AOE 11+소멸), 부트 시퀀스(선천 방어도 10+소멸) | A 유지 |
| 덱 압축(아키타입 시너지) | 순수(0코 최대 3장 소멸+소멸) | 사용자 판정 |
| 확정 서치 | 비밀 기술(덱에서 스킬 1장+소멸), 비밀 병기(공격 1장+소멸) | A 유지 (Rare) |

### 강등 대상 (소멸 + 조건부/무작위/니치)

- 무작위 효과: 단련 천재·연금술(포션)·랜덤 파워 → B
- 소수 덱만 유의미: 색 무지개(구체 덱), 신호 부스트(파워 덱), 플라크 캐논(상태이상 덱) → B
- 조건부 자원 의존: 연장(방어도 의존), 도유(희귀 카드 덱 의존) → B
- 페널티 교환: 패닉 버튼(30방어+2턴 방어금지) → B

## 2b. 3+코스트 페널티 (-0.5 티어)

3코 이상은 턴 구성을 제약(한 턴에 다른 카드 못 씀 경우 많음). **기본 -0.5 티어**.

### 유지 조건

| 정당화 | 예시 | 판정 |
|---|---|---|
| 파워 카드(영구 효과) | 에코 폼(매턴 첫 카드 2회), 창조적 AI(매턴 파워 획득), 영원의 갑옷(판금 12) | A 유지 |
| 원샷 보상 큰 대기 | 잭팟(30딜+3장 0코 보상), 지옥불(다음 턴까지 대량 딜) | A 유지 |
| 확정 AOE 성장 | 굴러가는 바위(매턴 피해 증가) | A 유지 |
| 연쇄 효과 | 재앙(매 공격마다 공격 카드 획득) | A 유지 |

### 강등 대상

- 3+코 + 단일 타겟 + 조건 없음: 단순 딜러 → B
- 3+코 + 시너지 의존(구체·플라즈마·별 등 재료 필요) → B
- 5+코 초고코: 예외적 시너지 없으면 B
- 3+코 + 소멸: 복합 페널티 (-1.5) → B 기본

## 3. 자해 트레이드 판정 (중요)

**자해 손실 >= 적 손실이면 등급 낮춤.** 단순 교환은 가치가 없음.

| 패턴 | 예시 | 판정 |
|---|---|---|
| 내 힘 -N, 적 힘 -N (동등 교환) | 운명 공유 base (힘 2 vs 2) | C (net 0) |
| 내 힘 -N, 적 힘 -(N+1) (미세 이득) | 운명 공유+ (힘 2 vs 3) | B |
| 내 힘/HP -N → 자원·드로우·딜 전환 | 사혈(HP 3→에너지 2), 제물(HP 6→에너지 2 + 드로우 3) | A~S (가치 전환) |
| 내 힘 -N, 매턴 지속 효과 | 우정(힘 2→매턴 에너지) | B (장기전 이득) |

**핵심**: 단순 손실-손실 교환은 의미 없음. 손실을 더 큰 가치로 전환하는 경우만 높은 등급.

## 4. 조건부 카드 판정 (티어 vs 점수 분리)

**핵심 원칙**: 조건부 카드는 **티어(잠재 가치)를 유지**하고, **현재 덱과 시너지가 없을 때 점수에서 감점**.

- 티어: 카드의 최대 잠재 가치 기반 (시너지 덱에서 얼마나 강한가)
- 점수: 지금 이 덱에서 얼마나 쓸만한가 → 시너지 카드 없으면 감점 (-8 ~ -12)

### 맞춤점수 페널티/보너스 트리거 (`ApplyConditionalSynergyPenalty`, phase-aware)

페이즈 판정: `isEarly` (층 ≤ 18 또는 덱 ≤ 12장), `isLate` (층 ≥ 35 또는 덱 ≥ 24장).

| 트리거 텍스트 | 필요 시너지 | 초반 감점 | 중반 감점 | 후반 보너스 (시너지 충족 시) |
|---|---|---|---|---|
| `종말을 부여할 때마다` | "종말" archetype 2장+ | **-15** | -12 | **+8 ~ +12** |
| `적이 중독을 보유` | POISON 태그/"중독" archetype 2장+ | -12 | -10 | +6 ~ +10 |
| `영혼을 사용할 때` | "영혼" archetype 2장+ | -12 | -10 | +6 ~ +10 |
| `골골이가 살아있/공격` | "골골이" archetype 2장+ | -12 | -10 | +6 ~ +10 |
| `파워 카드를 사용` | "파워" archetype 2장+ | -10 | -8 | +5 ~ +9 |
| `휘발성 카드` | "휘발성" archetype 2장+ | -12 | -10 | +6 ~ +10 |

**후반 보너스 공식**: `latebonus + min(4, synergyCount - 2)` — 시너지 카드가 많을수록 추가 보상.

- 시너지 2장 미만: 초반/중반 페널티로 감점. 태그 `덱 불일치` 부착.
- 시너지 2장+ AND 후반: 보너스 가점. 태그 `시너지` 부착.
- 시너지 2장+ AND 중반: 기본 아키타입/덱 테마 시너지가 이미 처리하므로 여기서 추가 없음.

**핵심 의미**: 종말/영혼/골골이 같은 **스케일링 축**은 초반에 덱 완성이 어렵고 후반에 진가 발휘 → phase-aware가 자연스러움.

### 티어 강등이 정당한 케이스 (위 페널티와 별개)

여전히 티어 자체를 낮춰야 하는 경우:

| 조건 유형 | 예시 | 판정 |
|---|---|---|
| 딜레이 (N턴 후 피해) | 폭탄 (3턴 뒤 40 AOE) | B (어떤 덱이어도 즉발 아님) |
| Hand-dependent colorless | 선조의 망치 | B (범용 손 조건) |
| Retain clog (하수) | 고동치는 도끼 base | D (모든 덱에서 부정) |
| 페널티 교환 | 패닉 버튼 (30방어+2턴 방어금지) | B |
| 즉사 페널티 | 포석+ (체력 종말) | C |

**원칙**: 덱 시너지로 해결 가능한 조건 → 티어 유지 + 점수 페널티. 어떤 덱에서도 해결 불가한 구조적 약점 → 티어 강등.

**단, 덱 아키타입과 직결되어 시너지 덱에서 최상위 → 큐레이션 고티어 유지**:
- 지옥불 (소멸 덱 필수): A 유지
- 수의/비눗방울 (종말/중독 시너지 전제): B 유지 + 비매치 시 점수 -10~-12
- 비밀 기술/비밀 병기 (Rare 서치): A 유지

## 5. 에너지 제공 카드 가산점

에너지 = **고밸류 템포 자원**. 티어 승격 + 맞춤점수 가중치 상향.

### 티어 승격 (큐레이션 CSV 판정)
**무조건** 에너지를 제공하는 카드는 +1 티어 승격 (최대 A까지).

확실한 승격 대상:
- 0-1코에 에너지 +1 이상 즉시 지급 (예: 터보 0코 +2, 사혈 0코 +2, 에너지 충격 1코 +2)
- 2-3코 파워로 매턴 +1 에너지 영구 (예: 화장용 장작, 영지)
- 지연이지만 대량 지급 (예: 부름+ 1코 다음 턴 에너지 3 + 소환 3 → A)

**조건부는 제외**:
- 처치 시 / 카드 사용 조건 / 덱 상태 조건 → B 유지

### 맞춤점수 보너스 (CardAdvisorService.Evaluate)
에너지 관련 키워드 매칭 시 **+10점** (일반 키워드 +5~7 대비 높음 — 고밸류 반영). 매칭 패턴:
- 한국어: `에너지를 얻`, `에너지 획득`, `에너지를 회복`, `마나`
- 아이콘: `energy_icon` 포함 + `얻`/`획득`

## 5a. 코스트 차등 점수 (맞춤점수)

1코와 2코, 2코와 3코 사이 간격을 넓혀 저코스트 우위 명확화:

| 코스트 | 기본 | 조건부 추가 |
|---|---|---|
| 0코 (X 제외) | **+12** | — |
| 1코 | **+8** | +3 (덱 저코스트 < deckSize/5) |
| 2코 | **0** | — |
| 3코 | **-6** | -6 추가 (고코 비중 높을 때) |
| 4+코 | **-12** | -6 추가 (고코 비중 높을 때) |

인접 간격: 0→1(4), 1→2(8), 2→3(6), 3→4(6). 2→3 간격은 시너지 없는 3코를 확실히 감점.

## 6. 희귀도 보정

| 희귀도 | 보정 |
|---|---|
| Basic | 감점. Strike/Defend류 → C 고정 |
| Common | 무조정. 평균 B |
| Uncommon | +0 ~ +0.5 (효과 좋으면 A 가능) |
| Rare | +0.5. 조건부 레어도 A까지 가능 |
| Ancient | **자동 조정 금지**. 스페셜 밸런스라 현재 티어 유지 |
| Curse/Status/Event/Token/Quest | 감사 제외 |

## 7. 보호 카드 (PROTECTED)

사용자가 명시적으로 설정한 티어는 자동 변경하지 않음. 현재 보호 목록:

```
CARD.FLASH_OF_STEEL (upgraded) = S
CARD.MASTER_OF_STRATEGY (upgraded) = S
CARD.SECRET_TECHNIQUE (upgraded) = S
CARD.DARK_SHACKLES (upgraded) = A
CARD.FUEL (upgraded) = A
CARD.THINKING_AHEAD (upgraded) = A
CARD.PURITY (upgraded) = B
CARD.BOLAS (upgraded) = B
CARD.THE_GAMBIT (upgraded) = C
CARD.THRUMMING_HATCHET (base) = D
CARD.MINION_SACRIFICE (base) = D
CARD.ULTIMATE_STRIKE (upgraded) = A  (사용자 언급 후 조정)
```

보호 목록 업데이트가 필요하면 사용자에게 확인.

## 8. 리젠트 별 메커니즘

리젠트 카드는 "별" 보조 자원이 있음:
- **별 생성** (`★을?\s*얻`): +0.5 (후속 카드 자원 보조)
- **별 소모** (`별을?\s*소모`): 유효 코스트 +N (소모한 별 개수만큼)
- **별 스케일링** (`별당`, `★ 수만큼`): +0.5 (조건부 확장)

## 9. 네크로바인더 소환 메커니즘

소환(`소환\s*\d+`)은 골골이에게 HP N을 부여하는 효과. **방어도와 동일한 효율 기준 적용** — 소환 N ≈ 방어도 N.

소환은 방어도 대비 두 가지 특성:
- **지속성**: 턴 넘어 유지(+) — 방어도와 달리 턴종료 시 초기화되지 않음
- **공격 보너스**: 골골이가 자동 공격(+)
- **취약성**: 골골이가 죽으면 리셋(-)

순효과로 방어도와 동급 취급.

| 코스트 | C | B | A |
|---|---|---|---|
| 0코 | 소환 2-3 | 소환 4-5 | 소환 6+ |
| 1코 | 소환 5-6 | 소환 7-9 | 소환 10+ |
| 2코 | 소환 8-10 | 소환 11-13 | 소환 14+ |
| 3+코 | 소환 12-14 | 소환 15-19 | 소환 20+ |

**하이브리드 환산** (소환 + 방어도): `소환N + 방어도M*0.7` → 블록 등가치
- 예: 끌어당기기 (2코 소환 4 + 방어도 7) = 4 + 7×0.7 ≈ 9 block-eq → B
- 예: 각성 (3코 소환 20 + 소멸) → 3코+소멸 복합 페널티(-1.5)로 소환 20 자체는 S 수치지만 B 기본. 골골이 덱 핵심이면 A까지.

**보존(`보존`) + 소환**: 매턴 재사용 가능 → **+1 티어**.
- 예: 박력(1코 보존 소환 3 + heal 5) → 매턴 소환 3 + 회복 5 = A
- 예: 박력+ (1코 보존 소환 5 + heal 7) → A

**소환 + 부가효과** (취약 부여, 드로우, 에너지):
- HIGH_FIVE (2코 골골이 AOE 11 + 취약 2) → 공격+디버프 = A upgrade
- NECRO_MASTERY (2코 소환 5 + 골골이 피해 반사) → 시너지 강력 = A upgrade

**조건부 골골이** (`골골이가 살아있다면`): -1 티어 (생존 조건 필수).
- 예: 뼛조각 (1코 AOE 9 + block 9, 골골이 생존 시) → B

**Basic 소환** (BODYGUARD): 시작 덱 = C 고정.

## 10. 하이브리드 카드 — 기본 효과 + 긍정 토큰 추가

1차 효과(딜/방어도)가 해당 코스트의 B 임계값 이상이고, 2차 효과로 **사용 가능한 긍정 토큰**을 뽑을 더미/손/버린 더미에 추가하면 **+1 티어**.

### 긍정 토큰 (드로우처럼 작동)
- 영혼 / 영혼+ (Necrobinder 공격 스포이더)
- 단도 / 단도+ (Silent shiv, 0코 1-2딜)
- 공허 (Defect void)
- 복사본 (카드 자기 복제)
- 카드 자체 드로우 (`카드를?\s*\d+장\s*뽑`)

### 부정 토큰 (페널티, 하이브리드로 치지 않음)
- 부상 (wound, unplayable)
- 어지러움 (dizzy)
- 점액투성이 (gunk)
- 상태이상 / 저주

### 승격 예시

| 카드 | 기본 | + 토큰 | 판정 |
|---|---|---|---|
| 무덤지기 | 1코 8 방어도 (B 임계) | + 영혼 1장 | **A** (`GRAVE_WARDEN`) |
| 무덤지기+ | 1코 11 방어도 (A 임계) | + 영혼 1장 | **A** |
| 약탈 / + | 1코 9/11 피해 (B/A) | + 영혼+ 1장 | **A** |
| 단절 / + | 2코 13/18 피해 (B/A) | + 영혼 3장(3곳 배치) | **A** |
| 혼 포획 / + | 1코 적 HP 3 감소 (작음) | + 영혼 3장(엔진) | **A** (핵심은 대량 토큰 생성) |
| 폼멜 타격 | 1코 9 피해 (B 임계) | + 카드 1장 뽑기 | **A** (`POMMEL_STRIKE`, 레퍼런스·ML 모두 S 판정 → 큐레이션 S 유지) |
| 폼멜 타격+ | 1코 10 피해 (A 임계) | + 카드 2장 뽑기 | **A** (드로우 2 = 실질 S급, 레퍼런스·ML 모두 S → 큐레이션 S 유지) |

### 제외 조건

1차 효과가 C 임계 미만이면 토큰 추가 있어도 **승격하지 않음**:
- 외투와 단도 (1코 6 방어도 + 단도 1-2): 6 방어도 = C 임계 → B 유지
- 선도 타격 (1코 3-6 피해 + 단도 2): 3-6딜 = D-C 임계 → B 유지

`사용 가능한` 핵심 — 뽑았을 때 아무 효과 없거나 부정 효과면 승격 금지.

## 11. 네크로바인더 종말 메커니즘

종말(`종말`)은 카운트다운 디버프(단일 타겟 기본). AOE가 아니므로 초반 가치 제한적. **종말 부여의 가치는 시너지 축에 의존**.

### DoT 발동 타이밍 비교 (중요)

| DoT | 발동 시점 | HP 감소 | 즉발성 판정 |
|---|---|---|---|
| **독(중독, poison)** | 내 턴 **종료 시** | O (실제 체력 감소) | **즉발성** — 적 턴 전에 적용 |
| **종말(doom)** | 적 턴 **종료 후** | O (처치 메커니즘) | **지연** — 적이 먼저 행동 |
| **불꽃(burn 등)** | 적 턴 종료 시 | O | 지연 |

광역(`모든 적에게`) 카드를 감정할 때:
- `모든 적에게 피해` → 직접 AOE: +6
- `모든 적에게 중독` → 독 AOE: **+5** (내 턴 끝에 HP 감소하므로 즉발에 가까움)
- `모든 적.*취약/약화` → 즉발 디버프: +5
- `모든 적에게 종말` → 종말 AOE: **+3** (적 턴 후 발동, 지연)

### 종말 카드 5분류

**중요**: 종말은 **적 턴 종료 후 발동**. "모든 적에게 종말 부여" = 광역 공격이 아님. 즉발 처치/디버프 같은 시너지 없이는 고평가 불가.

| 분류 | 특성 | 판정 |
|---|---|---|
| **ATTACK+DOOM** | 공격 자체로 가치 있음 (역병 타격, 사신의 모습, 최후의 시간) | 공격 효율대로 (B~A) |
| **ENGINE_DOOM** | 매턴 자동/여러 카드에 부여 (카운트다운, 망각) | 자동 누적이라 A 근처 |
| **AOE_DOOM (즉발 결합형)** | 모든 적 종말 + **즉시 처치/디버프** (종말의 날 = 체력 종말 이하 즉시 처치, 데스브링어 = AOE 약화 동반) | 즉발 효과 덕분에 B~A |
| **AOE_DOOM (단순 부여)** | 모든 적 종말만, 즉발 효과 없음 (부정 파동 base) | 광역 공격 아니므로 감점 → C |
| **SINGLE_DOOM** | 단일 타겟 슬로우 적용 (탈출할 수 없다 base) | C (조건부) |
| **SYNERGY_ONLY** | 종말 부여 "시" 트리거 (수의) | 종말 축 필수 → **C** |

### 즉발 결합 판별

"모든 적 종말 부여" 카드를 감정할 때:
- **즉발 처치**: `체력이 종말 이하`, `즉시 처치`, `처치합니다` → 즉발 효과 있음, 광역으로 취급
- **즉발 디버프**: `약화`, `취약` 부여 → 즉발 효과 있음
- **둘 다 없음**: 다음 턴 이후에 효과 → 광역 공격 취급 금지, C 기본

### 방어 + 종말 카드 구별 (사용자 지침)

**원칙**: 공격 + 종말은 딜로도 쓸 수 있지만, 방어 + 종말은 방어 가치가 떨어지고 종말 축 시너지에 의존 → 저평가.

예시:
- **수의 (SHROUD)**: 1코 파워, 종말 부여 시 방어도 2-3 → **C** (단독 발동 불가, 종말 덱 필수)
- **죽음의 문 (DEATHS_DOOR)**: 1코 방어도 6-7 + 종말 부여했다면 추가 방어 → B (평균값 적절, 시너지 덱에선 A)
- **부정 파동 (NEGATIVE_PULSE) base**: 1코 방어도 5 + AOE 종말 7 (즉발 없음) → **C** (광역 공격 아님, 방어만 단독 가치)
- **부정 파동+**: 방어도 6 + AOE 종말 11 → B (11 종말은 DoT 누적 유의미)
- **탈출할 수 없다 (NO_ESCAPE) base**: 1코 단일 종말 10 → **C** (초반 슬로우)
- **탈출할 수 없다+ **: 단일 종말 15 + 누적 보너스 → B

**판정 흐름**:
1. 종말 효과가 단독으로 가치 발휘 가능한가? (AOE / 매턴 자동 / 공격과 결합)
   - Yes → 일반 효율 기준 적용
   - No → **-1 티어** (시너지 전용)
2. 1차 효과(방어/공격)가 저수치(C 임계)이고 종말 효과가 조건부면 → C 고정

## 12. 사일런트 독 메커니즘

독은 **내 턴 종료 시 발동**하는 즉발성 DoT (종말과 달리 적 턴 전에 HP 감소). 단독 카드로도 가치 있지만, 스케일링·축적 시너지 덱에서 진가 발휘.

### 독 카드 분류

| 분류 | 특성 | 판정 |
|---|---|---|
| **ATTACK+POISON** | 공격 자체 가치 + 독 보너스 (독 찌르기) | 공격 효율대로 |
| **AOE_POISON (누적형)** | 매턴 자동 적용 (유해 연기, 부식성 파도) | B~A (엔진) |
| **PURE POISON** | 단일 타겟 독 부여 (살상용 독, 뱀의 이빨) | 소량(3~5)=C, 중량(7+)=B |
| **POISON_SCALER** | 독을 배가/추가 발동 (살포, 촉진제, 안개) | 독 축 핵심 → A |
| **CONDITIONAL_POISON** | 적이 독 보유 시에만 (비눗방울) | 단독 무효 → **C** (시너지 전제) |

### 핵심 원칙

- **독은 실드 관통**: 적 방어도를 무시하고 HP 직접 감소 → 일반 공격 대비 +4 맞춤점수 보너스 (`ApplyKoreanKeywordAnalysis` 후단)
- 독 AOE: **즉발성 DoT**이므로 종말 AOE와 달리 감점 없음 (+5 AOE 보너스)
- 조건부 발동(`적이 중독 보유 시`, `중독을 N번 부여할 때마다`): **티어는 유지**, 덱에 독 부여 카드 2장 미만이면 점수 -10
- 단순 소량 독 부여(1코 독 3~4): C 유지 (Poisoned Stab 같은 1코 6딜 보너스도 C)
- 중량 독(7+) 또는 보존 결합(뱀의 이빨+): B

### 예시

- **독 찌르기 (POISONED_STAB)**: 1코 6/8dmg + 독 3/4 → C/B (공격 C 임계 + 작은 독)
- **살상용 독 (DEADLY_POISON)**: 1코 단일 독 5/7 → C/B
- **뱀의 이빨 (SNAKEBITE) base/+**: 2코 보존 + 독 7/10 → C/B (보존 덕에 반복 가능)
- **비눗방울 (BUBBLE_BUBBLE) base**: 1코 적 중독 보유 시 독 9 → **C** (단독 무효)
- **유해 연기 (NOXIOUS_FUMES)**: 1코 파워, 매턴 AOE 독 2/3 → B (엔진)
- **부식성 파도 (CORROSIVE_WAVE)**: 1코 Rare, 드로우 시 AOE 독 2/3 → B/A (드로우 엔진)
- **안개 (HAZE)**: 3코 교활 + AOE 독 4/6 → B (3코지만 교활 가치)
- **촉진제 (ACCELERANT)**: 1코 파워, 독 1번/2번 추가 발동 → B/A (독 덱 핵심)
- **독성 무장 (ENVENOM)**: 2코 파워, 공격 시 독 1/2 부여 → B/A (엔진)
- **신기루 (MIRAGE)**: 1코 AOE 독량만큼 방어도 + 소멸 → B/A (독 축 의존)

## ML 모델 2차 검증 (TierModelScore)

`card_tier_data.csv`에는 오프라인 학습된 ML 모델의 예측값이 이미 포함되어 있다.
**규칙 판정 후** ML 점수와 대조해 이상 신호를 감지한다.

### CSV 컬럼 의미

| 컬럼 | 내용 |
|---|---|
| `TierModel` | ML 예측 등급 (D/C/B/A/S) |
| `TierModelScore` | 연속 점수 0.00~4.00 (D=0, C=1, B=2, A=3, S=4) |
| `TierModelDelta` | Tier − TierModel 차이 (`+1`=ML이 한 단계 높게 예측) |
| `TierSource` | `json`=레퍼런스로 확정, `auto`=ML 미덮어쓰기 |

### TierModelScore 구간 해석

| 점수 | 의미 |
|---|---|
| 0.00 ~ 0.80 | D 영역 |
| 0.80 ~ 1.80 | C 영역 |
| 1.80 ~ 2.80 | B 영역 |
| 2.80 ~ 3.60 | A 영역 |
| 3.60 ~ 4.00 | S 영역 |

### 사용 방법 — 판정 흐름

```
1. 규칙 기반 판정(섹션 1~12) → 제안 등급 결정
2. CSV에서 해당 카드의 TierModelScore 조회
3. 비교:
   a. 규칙 제안 = TierModel → 양쪽 일치, 높은 신뢰도로 채택
   b. 규칙 제안 ≠ TierModel (1 tier 차이) → 규칙 우선, 단 근거 재확인
   c. 규칙 제안 ≠ TierModel (2+ tier 차이) → 강한 불일치 신호, 카드 설명 재검토
4. TierModelScore가 경계에 걸릴 때 (예: 2.75 = B/A 경계)
   → 가까운 구간을 우선 참고해 규칙 판정이 맞는지 확인
```

### 신뢰도 규칙

- **`TierSource=json` 카드**: 레퍼런스(구글 시트) 기반 확정값. ML 점수가 달라도 기존 Tier 우선.
- **`TierSource=auto` 카드**: ML 미반영 상태. ML 점수가 규칙 판정과 일치하면 더 강하게 채택.
- **신규 카드** (CSV에 없거나 `TierModelScore` 비어있음): ML 점수 없음 → 규칙 판정만 사용.

### ML 점수 갱신이 필요한 경우

새 카드가 추가되었거나 `TierModelScore`가 비어있으면:
```bash
python scripts/build_card_weights.py
```
→ `card_tier_data.csv`의 `TierModel`, `TierModelScore`, `TierModelDelta` 컬럼 갱신.
재학습까지 원하면 `python scripts/rebuild_tiers.py` (수 분 소요).

## 워크플로우

### 오딧 실행
1. `card_tier_data.csv` 로드. `Description`에서 dmg/blk 파싱.
2. 각 카드에 순수 효율 기준점 계산 → 부가 효과로 ±조정 (섹션 1~12).
3. **ML 2차 검증**: `TierModelScore` 대조 — 2+ tier 불일치 시 재검토.
4. 현재 Tier와 비교. **gap ≥ 2면 자동 승격/강등**, gap 1은 수동 검토.
5. PROTECTED / Ancient / 특수 희귀도는 제외.
6. 사용자에 변경 목록 제시 → 승인 후 CSV 수정.

### 변경 적용
- CSV `Tier` 컬럼만 수정. 다른 컬럼 건드리지 않음.
- 수정 후 `dotnet build Sts2CardAdvisor.csproj` 실행. 변경된 CSV가 DLL에 embedded.
- 분포 로그: `Counter(r['Tier'] for r in rows)`

### S 등급 주의사항
- 자동 승격으로 S 부여하지 않음. **큐레이션·사용자 명시만**.
- ML이 S 예측(`TierModelScore ≥ 3.60`)해도 A로 캡 — 최종 S 확정은 사용자 명시 필요.
